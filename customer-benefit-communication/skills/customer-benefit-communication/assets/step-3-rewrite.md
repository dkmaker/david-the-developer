# Step 3: Rewrite with Benefits

## The Benefit Bridge Formula

```
[What we did/do] → [so that / which means] → [what you gain]
```

## Before → After Examples

| Before (technical) | After (benefit-driven) |
|---|---|
| "We upgraded your firewall." | "We strengthened your security, so you're better protected against the latest threats." |
| "Migration to cloud completed." | "Your systems now run in a certified EU data centre. Higher uptime, automatic backup, no hardware worries." |
| "MFA has been enabled for all users." | "Your employees now log in with an extra layer of security. Even if a password is stolen, no one can get in." |
| "47 tickets resolved this month." | "We resolved 47 requests with an average response time of ~12 min — so your team could get back to work fast." |
| "The backup job failed; we restarted it." | "We caught a backup issue and fixed it. Your data is safe — everything is running normally." |

## Tone by Recipient

| Recipient | Opener style | Focus |
|-----------|-------------|-------|
| **CEO / Owner** | "What this means for your business…" | ROI, growth, risk |
| **Office manager / admin** | "In practice, this means your day-to-day…" | Ease, fewer headaches |
| **IT responsible** | "Technically, this gives you…" | Stability, compliance |
| **Finance responsible** | "From a cost perspective, this means…" | Cost, budget |
| **End user** | "For you, this means…" | "It just works" |

## Language Rules

- Match input language exactly (Danish in → Danish out, English in → English out)
- Avoid jargon unless writing to an IT-responsible recipient
- Use "you / your" — the customer is always the subject
- If the user explicitly requests a specific output language, use that instead
- Keep sentences short. One benefit per sentence is more powerful than three crammed together.
